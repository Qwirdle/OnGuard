On the internet, all data must be handled carefully. The problem with data, however, is that by making something more secure, you compromise other aspects of it.

This is called the “CIA Triad”. It consists of 3 components:

  • Confidentiality

    • The guarantee that only people meant to see it will

  • Integrity

    • The guarantee that the data will remain unchanged

  • Availability

    • The guarantee that the data will be available at any given moment

It is completely unrealistic to completely fill even one of these parts of the triad. To make it perfectly confidential you would need to meet up somewhere secret and exchange SSDs of data physically. To raise its integrity, you would need to make it so no one could ever switch or modify any contents. Finally, to make it available to all, a QR code that links to a separate download link (in case one goes down) must be placed at every lamp post in the world.

The realistic tradeoffs for each type, however, are that:

  • If data must be confidential, it should be encrypted and locked behind passwords or accounts, removing availability. This may also remove integrity since some encryption ciphers completely change data if even one character is changed.

  • If the data must be available, it will sacrifice confidentiality due to how easy it will be to find, and integrity because hackers would be more likely to find vulnerabilities.

  • If the data must not be changed, it will need multiple backups and confirmations before changes, which would take much storage and remove both availability and confidentiality

Summary

  • Follow the CIA Triad

    • Every bit of data needs at least some of each element to not make it unworkable

  • Most data will need medium to high levels of encryption. This includes

    • Passwords

    • Medical records

    • Many documents/contracts

  • Attempting to fill out everything in the CIA Triad is unrealistic
